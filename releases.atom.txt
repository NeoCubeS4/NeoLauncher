    <content type="html">&lt;h2&gt;Release v2.0.6&lt;/h2&gt;
&lt;p&gt;Most user facing strings are now fully localized. The default translation file for English is included in app/assets/lang/en_US.toml. Data points that nearly every fork will change are extracted out in _custom.toml. This will hopefully make it easier for forks to create and maintain translations. At the moment translations will not be included in the base repository since it can be difficult to maintain them.&lt;/p&gt;
&lt;p&gt;The minecraft.net status check has been removed. Pinging the website with a simple GET request no longer successfully returns, which had caused the service to appear constantly offline. The status check for this service is not particularly useful, so it has been removed.&lt;/p&gt;
&lt;p&gt;A number of additional fixes and upgrades are included.&lt;/p&gt;
&lt;p&gt;Electron has been upgraded to v27.&lt;/p&gt;

    <title>Release v2.0.4 - Harden Java Discovery Logic, Electron Upgrade</title>
    <content type="html">&lt;h2&gt;Release v2.0.4&lt;/h2&gt;
&lt;p&gt;This update fixes a few issues with the Java discovery to make it more fault tolerant. Namely fixes the following:&lt;/p&gt;
&lt;ol&gt;
&lt;li&gt;Only check against the standard &lt;code&gt;java.version&lt;/code&gt; property to avoid obscure formats by different vendors. This also means that the build number is no longer collected.&lt;/li&gt;
&lt;li&gt;If a JVM version format is not valid, warn and skip. Do not throw an error.&lt;/li&gt;
&lt;li&gt;Add error handling around -XshowSettings invocation. This is to prevent derailing the Java discovery process in the event a nonexistent path is passed or an unrelated error occurs during command invocation.&lt;/li&gt;
&lt;/ol&gt;
&lt;h3&gt;Fixes/Upgrades&lt;/h3&gt;

&lt;li&gt;&lt;a class=&quot;commit-link&quot; data-hovercard-type=&quot;commit&quot; data-hovercard-url=&quot;https://github.com/dscalzi/HeliosLauncher/commit/d89b270c208f4d40dc7ff0844dee663d2d085f53/hovercard&quot; href=&quot;https://github.com/dscalzi/HeliosLauncher/commit/d89b270c208f4d40dc7ff0844dee663d2d085f53&quot;&gt;&lt;tt&gt;d89b270&lt;/tt&gt;&lt;/a&gt; Harden Java discovery logic, upgrade to Electron 24.&lt;/li&gt;

    <title>Release v2.0.3 - Improve Error Message for Users w/out a MC Profile</title>
    <content type="html">&lt;h2&gt;Release v2.0.3&lt;/h2&gt;
&lt;p&gt;This update improves the error message for users who have not set up a MC profile for their Microsoft account. This can happen if the user purchases the game (either through minecraft.net or Xbox Game Pass) and does not set up a profile, or if the user does not own the game at all. An improved error message will now notify the user of this.&lt;/p&gt;

    <title>Release v2.0.2 - Make Version Regex More Lenient</title>
    <content type="html">&lt;h2&gt;Release v2.0.2&lt;/h2&gt;
&lt;p&gt;This is a bugfix for &lt;a href=&quot;https://github.com/NeoCubeS4/NeoLauncher/releases/tag/v2.0.0&quot;&gt;release v2.0.0&lt;/a&gt;. Some Java distributions (Oracle) format their runtime versions differently than others. This update makes the regex more lenient to accomodate.&lt;/p&gt;
